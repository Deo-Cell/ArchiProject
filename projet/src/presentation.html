Filtrage Dynamique — Expression Trees

Type Syntaxe URL Expression LINQ Exemple concret
Exact ?name=Margherita x.Name == "Margherita" 1 seule valeur
Multiple ?name=Margherita,Reine list.Contains(x.Name) Plusieurs valeurs (OU)
Range ?price=[10,12] x >= 10 && x <= 12 Intervalle
GreaterThan ?price=[13,] x.Price >= 13 Supérieur ou égal
LessThan ?price=[,10] x.Price <= 10 Inférieur ou égal

Versioning — URL Segment

Principe
  • Version dans l'URL : /api/v1/pizzas
  • Stratégie : UrlSegmentApiVersionReader
  • Version par défaut : v1.0
  • Évolution sans casser le contrat client
Configuration centralisée
  • ApiVersionConfig dans la Library
  • AssumeDefaultVersionWhenUnspecified
  • ReportApiVersions = true
  • Headers de réponse enrichis
Avantages
  • Rétro-compatibilité garantie
  • Plusieurs versions en parallèle
  • /api/v1/ et /api/v2/ coexistent
  • Swagger documente chaque version

Défis Techniques Rencontrés

decimal vs double — SQLite
  • SQLite refuse ORDER BY sur decimal
  • Erreur silencieuse, tri incohérent
  • Solution : Migration decimal → double
  • Appliquée sur tous les modèles
SQLite limitation Migration EF Core
Expression Trees dynamiques
  • Construire du LINQ sans connaître T
  • Conversion string → double / DateTime
  • Solution : Expression.Convert()
  • Gestion silencieuse des erreurs
Runtime types System.Linq.Expressions
Projection & EF Core
  • EF Core ne traduit pas Dictionary en SQL
  • Crash si appliqué sur IQueryable
  • Solution : Projection après pagination
  • Exécutée en mémoire (50 items max)
EF Core limitation Post-query projection
